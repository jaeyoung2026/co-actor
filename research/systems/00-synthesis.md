# 시스템 상상 종합 — 8개 결과물에서 1개의 시스템으로

> 3개 핵심 원리(약속, 주의력, 관계) × 4관점 × 2에이전트 = 8개 시스템 설계
> 이 문서는 8개를 비교 분석하고 공통 구조를 추출한다.

---

## 8개 시스템 일람

| # | 이름 | 관점 | 에이전트 | 핵심 메타포 |
|---|------|------|---------|-----------|
| 1 | Covenant (언약) | 순수 과제 | 브로콜리 | 언약 등록부 + 주의력 탐침 + 관계 원장 |
| 2 | Mycelial Dialogue System | 생태학자 | 브로콜리 | 토양(약속) + 균사체(주의력) + 수관 경계(관계) |
| 3 | (이름 없음) | 첫 번째 원리 | 브로콜리 | Promise Register + Attention Budget + Agency Gradient |
| 4 | Homeostatic Loop (항상성 루프) | 시스템 사상가 | 브로콜리 | 항상성 + 시상 필터 + 호혜적 교환 |
| 5 | Promise Loop | 순수 과제 | 코덱스 | Promise Ledger + Attention Frame + Relationship Memory |
| 6 | Tidal Dialogue Ecology (조간대) | 생태학자 | 코덱스 | 기후층 + 근권층 + 종간층 + 퇴적층 |

| 8 | PRR Loop | 시스템 사상가 | 코덱스 | 유전자 발현 조절 + 에너지 분포 + 반복 상호작용 |

## 공통 구조 (8개 모두 수렴)

### 1. 3층 순환 — 예외 없음
모든 설계가 약속→주의력→관계→약속의 순환 구조를 채택했다. 3개의 독립 모듈이 아니라 하나의 순환 시스템이다.

### 2. 약속 = 정체성의 불변 조건
- 약속은 프롬프트가 아니라 **검증 가능한 명제**로 선언된다
- 두 종류: **항구적 약속**(서비스 정체성) + **국소 약속**(대화 국면별, 사용자와 합의)
- 모델/프롬프트가 바뀌어도 약속이 같으면 코르카는 같은 코르카다

### 3. 주의력 = 유한 슬롯의 명시적 관리
- 컨텍스트 전체를 넣지 않는다. **소수의 슬롯**(4-5개)에 명시적으로 배치
- 핵심 기술은 더 긴 컨텍스트가 아니라 **턴 단위 주의력 편성**
- 슬롯 구성 (합의): `현재 질문`, `활성 약속`, `핵심 증거/탐색 갈래`, `관계 신호`
- 엔트로피 계측: 대화가 길어질수록 분산도를 측정하고 재정립 트리거

### 4. 관계 = 최종 품질 게이트
- 품질의 최종 척도: "좋은 답변인가"가 아니라 **"다음 턴에도 함께 일하고 싶은가"**
- 핵심 지표: 주도권 균형, 회복 이력, 경계 일관성(예측 가능성)
- 사실 기억보다 **협력 기억**을 우선 저장

### 5. 실패 분류 = 어느 층에서 깨졌는가
- 약속 위반 → 프롬프트/가드레일 수정
- 주의력 실패 → 컨텍스트 구성 수정
- 관계 실패 → 약속이 불완전하다는 신호 → 새 약속 후보 생성

### 6. 평가 단위의 전환
- 기존: 턴 단위 출력 채점
- 새 시스템: **순환 단위** — 약속 준수 → 주의 배치 적절성 → 관계 영향 순으로 평가

## 독자적 통찰 (개별 설계에서만 나온 것)

| 통찰 | 출처 | 가치 |
|------|------|------|
| **Agency Gradient** — 수행↔제안↔질문의 연속적 기울기. 확신이 낮을수록 질문 쪽으로 | #3 첫번째원리 브로콜리 | 이산적 모드 전환의 대안. 베이지안 추론과 연결 |
| **Crown Shyness** — 수관이 서로 닿지 않는 현상. 주도권 경계의 자연스러운 메타포 | #2 생태학자 브로콜리 | 관계 설계의 직관적 비유 |
| **시상 필터(Thalamic Filter)** — 피질에 도달하는 신호를 선별하는 게이트 | #4 시스템사상가 브로콜리 | 주의력 슬롯의 신경과학적 근거 |
| **퇴적층** — 당장 쓰지 않지만 이후 떠오를 논점·가설·미해결 질문 | #6 조간대 코덱스 | 장기 기억과 주의력의 분리 |
| **만족이 아니라 약속 이행이 목표** — 사용자가 만족해도 약속 위반이면 실패 | #4 항상성루프 브로콜리 | sycophancy 방지의 구조적 장치 |
| **협력 기억 > 사실 기억** — 어디서 합의/충돌했는지가 무엇을 찾았는지보다 중요 | #5 Promise Loop 코덱스 | 관계 축적의 핵심 |

## 제안: lighthouse를 위한 통합 시스템

### 이름: **Promise-Attention-Relationship Loop (PAR Loop)**

### 구조

```
┌─────────────────────────────────────────────┐
│                  PAR Loop                    │
│                                             │
│   ┌──────────┐    ┌──────────┐    ┌────────┐│
│   │ Promise  │───→│Attention │───→│Relation││
│   │ Registry │    │ Frame    │    │ Ledger ││
│   └────┬─────┘    └──────────┘    └───┬────┘│
│        │                              │      │
│        └──────── revision ────────────┘      │
│                                             │
│   ┌─────────────────────────────────────┐   │
│   │        Loop Audit (매 턴)            │   │
│   │  약속 지켰나? → 주의력 적절했나?     │   │
│   │  → 관계 강화/손상? → 약속 갱신       │   │
│   └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### Promise Registry
- **항구적 약속**: 코르카의 정체성. "연구자의 판단을 대체하지 않는다", "근거 없는 단정을 하지 않는다"
- **국소 약속**: 대화 국면마다 사용자와 합의. "이번 탐색에서는 방법론 한계를 집중적으로 본다"
- 약속 = 검증 가능한 술어. 위반은 실시간 감지.

### Attention Frame (4슬롯)
1. 현재 질문 / 사용자 의도
2. 활성 약속 (위반 감시용)
3. 핵심 증거 / 탐색 갈래
4. 관계 신호 (주도권 균형, 감정 단서)

- 새 정보 유입 시 기존 슬롯과 경쟁. 밀려난 항목은 **퇴적층**(압축본)으로.
- 엔트로피 임계값 초과 시 "지금 우리 어디에 있지?" 재정립 트리거.

### Relationship Ledger
- **주도권 균형**: 최근 N턴의 방향 결정자 비율
- **회복 이력**: 오류 후 교정 수용 속도/깔끔함
- **경계 일관성**: "이건 내가 판단할 수 없다"의 패턴 일관성
- **Agency Gradient**: 확신도에 따라 수행↔제안↔질문을 연속 조절

### Loop Audit
매 턴 후 3개 질문:
1. 약속을 지켰는가? → 위반이면 어떤 약속을?
2. 주의력이 적절했는가? → 오배치면 어떤 슬롯이?
3. 관계를 강화했는가? → 손상이면 어떤 지표가?

결과 → 약속 갱신 또는 주의력 재편성.

### 실패 분류
| 층 | 증상 | 대응 |
|----|------|------|
| 약속 위반 | 금지 행동 수행, 경계 불일치 | 프롬프트/가드레일 수정 |
| 주의력 실패 | 관련 정보 누락, 의도 오인 | 컨텍스트 구성 수정 |
| 관계 실패 | 주도권 침해, 회복 불가 | 약속 자체가 불완전 → 새 약속 후보 |

## 핵심 차이: 이것은 무엇을 재정의하는가

**기존:** "좋은 답변을 만들었는가?" — 출력 중심, 턴 단위 채점
**PAR Loop:** "좋은 동료로 있었는가?" — 관계 중심, 순환 단위 진단

기존이 **행동 정책 엔지니어링**이라면, PAR Loop는 **대화 존재 방식 자체의 엔지니어링**이다.
