# Promise Loop — 순수 과제 × 코덱스

## 핵심 정의
코르카의 대화 품질을 "좋은 답 생성"이 아니라, 약속을 갱신하며 관계를 축적하는 순환 시스템으로 다루는 설계.

## 구조

### 1. Promise Ledger
대화 품질을 규칙이 아니라 검증 가능한 약속 묶음으로 선언한다.
각 약속: `무엇이 참이어야 하는가 / 무엇이 위반인가 / 무엇으로 검증하는가`

### 2. Attention Frame
매 턴마다 소수의 요소만 전면에 배치. 4칸: `현재 질문`, `활성 약속`, `살아있는 탐색 갈래`, `관계 신호`.
긴 세션에서 품질 저하 = 능력 부족이 아니라 주의력 오염.

### 3. Relationship Memory
사실 기억보다 먼저 협력 기억을 남긴다.
`어디서 합의했는가 / 어디서 충돌했는가 / 어떤 제안 방식이 사용자의 주도권을 살렸는가`

### 4. Loop Review
`약속 준수 → 주의 배치 적절성 → 관계 영향` 순으로 보고, 약속 자체를 갱신.
품질 엔지니어링 단위 = "프롬프트 수정"이 아니라 순환의 재설계.

## 핵심 차이
모델이 바뀌어도 약속이 같으면 코르카는 같은 코르카다. 품질은 프롬프트 문구가 아니라 무슨 약속을 들고, 무엇에 주의하고, 어떤 관계를 남기는가로 결정된다.
