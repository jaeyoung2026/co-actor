# Covenant (언약) — 순수 과제 × 브로콜리

## 핵심 구조

세 개의 층과 하나의 순환.

### 1층: Promise Registry (약속 등록부)
코르카가 연구 동료로서 지켜야 할 언약을 검증 가능한 명제로 등록한다. 구현 독립적.

- **경계 언약**: 위반 시 관계가 손상되는 것 (결정 대행 금지, 출처 없는 단정 금지)
- **지향 언약**: 충족도가 연속적인 것 (탐색 범위의 적절성, 맥락 요약의 충실도)

### 2층: Attention Probe (주의력 탐침)
매 턴을 채점하지 않는다. 코르카의 주의력이 올바른 곳에 있는가를 탐침한다.

- 주의력 지도: "이 턴에서 코르카가 실제로 참조/반영한 정보가 무엇인가"의 의미적 지도
- 사용자 관심 지도와 대조
- 엔트로피 계측: 분산 임계값 초과 시 재정립 신호
- 단계 전이: 미해소 질문의 감소율 + 새 정보의 한계 기여도로 결정

### 3층: Relationship Ledger (관계 원장)
관계의 건강도를 추적한다.

- 주도권 균형: 코르카 vs 사용자 방향 결정 비율
- 회복 이력: 교정 수용 속도/깔끔함
- 경계 예측 가능성: "판단할 수 없다"의 패턴 일관성

### 순환: Covenant Cycle
```
Promise Registry → 이 턴에서 어떤 언약이 활성화되는가?
    ↓
Attention Probe → 활성 언약에 필요한 정보에 주의력이 배치되어 있는가?
    ↓
Relationship Ledger → 이 턴의 결과가 관계를 강화했는가, 약화했는가?
    ↓
Promise Registry → 관계 피드백을 기반으로 언약을 재해석/보완한다
```

### 실패 분류
- 약속 위반: 언약 자체를 어겼다 → 프롬프트/가드레일 수정
- 주의력 실패: 언약은 알았지만 관련 정보를 놓쳤다 → 컨텍스트 구성 수정
- 관계 실패: 정보도 맞고 언약도 지켰지만 관계가 손상됐다 → 새 언약 후보 생성

### 기존과의 차이
Covenant는 "좋은 답변을 만들었는가"가 아니라 "좋은 동료로 있었는가"를 묻는다.
